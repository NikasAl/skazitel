# Roadmap Сказителя

## Стратегия

1. **Сначала Chrome-плагин** — быстрее разрабатывается, проще итерировать, мгновенно доступен для тестирования на себе
2. **Отладка функционала** — проходим все типы упражнений, настраиваем промпты, проверяем UX
3. **Порт на Android** — ориентируемся на проверенный плагин, переносим концепт в Kotlin + Compose
4. **Публикация** — RuStore (Android), Chrome Web Store (плагин)

## Фаза 3: Все типы упражнений + дрили ✅

**Цель:** Реализовать все типы упражнений (8 творческих + 4 дрили).

### Творческие упражнения (8 типов)

- [x] Ритмический тренажёр (`rhythm`) — промпты генерации + проверки, 2 встроенных
- [x] Рифмический конструктор (`rhyme`) — промпты генерации + проверки, 1 встроенное
- [x] Метафорический мост (`metaphor`) — промпты генерации + проверки
- [x] Ограничение как искусство (`constraint`) — промпты генерации + проверки
- [x] Деконструкция мастера (`deconstruction`) — промпты генерации + проверки
- [x] Фонетический этюд (`phonetics`) — промпты генерации + проверки
- [x] Перевод прозы в поэзию (`prose_to_poetry`) — промпты генерации + проверки
- [x] Анти-клише (`anti_cliche`) — промпты генерации + проверки

### Дрели — базовые навыки (4 типа, выбор из вариантов, без LLM-проверки)

- [x] Силлабический счётчик (`syllable_count`) — мгновенная проверка, 2 встроенных
- [x] Определение ударений (`stress_pattern`) — мгновенная проверка, 1 встроенное
- [x] Подбор рифмы (`rhyme_match`) — мгновенная проверка, 1 встроенное
- [x] Конструктор строки (`line_builder`) — мгновенная проверка, 1 встроенное

### Выбор типа упражнения

- [x] Пользователь может выбрать любой из 12 типов
- [x] Кнопка «Случайный тип» для случайного упражнения
- [x] Промпты масштабируются по сложности (getDifficultyParams)

### Обработка ошибок LLM

- [x] Авто-распаковка вложенного JSON (ключи: review, result, analysis, feedback и др.)
- [x] Нормализация flat-полей (rhythmScore → scores.rhythm, rhythm → scores.rhythm)
- [x] REVIEW_JSON_FORMAT — явные примеры формата в каждом промпте проверки
- [x] Фоллбэк на встроенные упражнения при ошибках API/LLM
- [x] Сохранение всех упражнений (включая fallback) в БД
