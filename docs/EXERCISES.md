# Система упражнений Сказителя

## Обзор

Упражнения — основа Сказителя. Каждый тип тренирует конкретный навык стихосложения. Упражнения генерируются через LLM на основе активной темы пользователя, либо берутся из встроенного набора (офлайн-режим).

### Типы упражнений

| Категория | Кол-во | Особенности |
|-----------|--------|------------|
| **Творческие** | 8 | Свободный ввод текста, проверка через LLM |
| **Дрели** | 4 | Выбор из вариантов, мгновенная проверка (без LLM) |
| **Итого** | **12** | |

### Цикл взаимодействия (творческие)

```
1. Пользователь выбирает тип упражнения
   ↓
2. LLM генерирует упражнение на основе активной темы + сложности
   (или используется встроенное упражнение)
   ↓
3. Пользователь выполняет задание (вводит текст)
   ↓
4. LLM проверяет ответ → ExerciseReview (scores, strengths, weaknesses)
   ↓
5. Фидбек + XP + обновление стрика/уровня
   ↓
6. Ответ автоматически сохраняется как стих в разделе «Мои стихи»
```

### Цикл взаимодействия (дрели)

```
1. Пользователь выбирает тип дрели
   ↓
2. LLM генерирует задание (или встроенное)
   ↓
3. Пользователь выбирает вариант ответа
   ↓
4. Мгновенная проверка по correctIndex (без LLM)
   ↓
5. Показ результата: «Верно» / «Неверно» + пояснение + XP
```

## Творческие упражнения

### 1. Ритмический тренажёр (`rhythm`) ✅

**Навык:** метрическое чутьё, чувство размера, подсчёт слогов.

**Механика:** Пользователь получает прозаический текст и должен переложить его в заданный стихотворный размер.

**Реализация:**
- ✅ Промпт генерации (`rhythmGeneratePrompt`)
- ✅ Специализированный промпт проверки (`rhythmReviewPrompt`) — с параметрами meter, syllableCount, expectedLines
- ✅ 2 встроенных упражнения (сложность 2 и 3)

**Пример:**
```
Задание: Перепиши этот текст ямбом:
«Дождь барабанит по крыше, ветер качает деревья»
Твой ответ должен содержать 4 строки по 8 слогов.
```

---

### 2. Рифмический конструктор (`rhyme`) ✅

**Навык:** поиск рифм, построение схем рифмовки.

**Механика:** Ключевые слова + схема рифмовки → составить четверостишие.

**Реализация:**
- ✅ Промпт генерации (`rhymeGeneratePrompt`)
- ✅ Специализированный промпт проверки (`rhymeReviewPrompt`) — с keywords, rhymeScheme
- ✅ 1 встроенное упражнение (сложность 2)

**Пример:**
```
Ключевые слова: окно, стекло, туман, обман
Схема рифмовки: abab
Составь четверостишие используя ВСЕ эти слова.
```

---

### 3. Метафорический мост (`metaphor`) ✅

**Навык:** построение неочевидных метафор.

**Механика:** Набор визуальных образов → написать двустишие на каждый с метафорической связью.

**Реализация:**
- ✅ Промпт генерации (`metaphorGeneratePrompt`)
- ✅ Специализированный промпт проверки (`metaphorReviewPrompt`) — с images, targetEmotion
- ❌ Нет встроенных упражнений

---

### 4. Ограничение как искусство (`constraint`) ✅

**Навык:** работа с формальными ограничениями.

**Механика:** Жёсткие ограничения (количество слов, части речи, времена глаголов и т.д.).

**Реализация:**
- ✅ Промпт генерации (`constraintGeneratePrompt`)
- ✅ Общий промпт проверки (`genericReviewPrompt`)
- ❌ Нет встроенных упражнений

---

### 5. Деконструкция мастера (`deconstruction`) ✅

**Навык:** анализ поэтических приёмов.

**Механика:** Отрывок известного стихотворения → написать свой вариант, используя те же приёмы.

**Реализация:**
- ✅ Промпт генерации (`deconstructionGeneratePrompt`)
- ✅ Общий промпт проверки (`genericReviewPrompt`)
- ❌ Нет встроенных упражнений

---

### 6. Фонетический этюд (`phonetics`) ✅

**Навык:** фонетическая музыкальность, звукопись.

**Механика:** Написать четверостишие с преобладанием определённых звуков.

**Реализация:**
- ✅ Промпт генерации (`phoneticsGeneratePrompt`)
- ✅ Общий промпт проверки (`genericReviewPrompt`)
- ❌ Нет встроенных упражнений

### 7. Перевод прозы в поэзию (`prose_to_poetry`) ✅

**Навык:** упаковка смысла в форму.

**Механика:** Прозаический текст → найти поэтические элементы и превратить в стихи.

**Реализация:**
- ✅ Промпт генерации (`proseToPoetryGeneratePrompt`)

## Дрели — базовые навыки

Дрели — упражнения с выбором из вариантов. Проверяются мгновенно (по `correctIndex`), без вызова LLM. Работают без API-ключа.

### 9. Силлабический счётчик (`syllable_count`) ✅

**Навык:** подсчёт слогов в слове или строке.

**Реализация:**
- ✅ Промпт генерации (`syllableCountGeneratePrompt`)
- ✅ Мгновенная проверка через `checkDrillAnswer()`
- ✅ 2 встроенных упражнения

**Примеры встроенных:**
- «Сколько слогов в слове «золотой»?» (3)
- «Сколько слогов в слове «вечерний»?» (4)

---

### 10. Определение ударений (`stress_pattern`) ✅

**Навык:** определение схемы ударений и стихотворного размера.

**Реализация:**
- ✅ Промпт генерации (`stressPatternGeneratePrompt`)
- ✅ Мгновенная проверка через `checkDrillAnswer()`
- ✅ 1 встроенное упражнение

**Пример встроенного:**
- «Мороз и солнце; день чудесный» → хорей

---

### 11. Подбор рифмы (`rhyme_match`) ✅

**Навык:** поиск точной рифмы.

**Реализация:**
- ✅ Промпт генерации (`rhymeMatchGeneratePrompt`)
- ✅ Мгновенная проверка через `checkDrillAnswer()`
- ✅ 1 встроенное упражнение

**Пример встроенного:**
- «Какое слово рифмуется с «зима»?» (дума)

---

### 12. Конструктор строки (`line_builder`) ✅

**Навык:** сборка строки с правильным ритмом из слов.

**Реализация:**
- ✅ Промпт генерации (`lineBuilderGeneratePrompt`)
- ✅ Мгновенная проверка через `checkDrillAnswer()`
- ✅ 1 встроенное упражнение

**Пример встроенного:**
- Rearrange words into iambic line

## Система сложности (1-10)

### Группы уровней

| Уровни | Название | Характеристика |
|--------|----------|---------------|
| **1-2** | Освоение формы | Простые рифмы, базовые размеры (4-стопный ямб), много подсказок |
| **3-4** | Развитие техники | Составные рифмы, чередование размеров, меньше подсказок |
| **5-6** | Мастерство формы | Сложные размеры, метафорические темы, жёсткие ограничения |
| **7-8** | Свобода в рамках | Внутренние рифмы, полиритмия, минимум указаний |
| **9-10** | Создание нового | Эксперименты с формой, философские темы |

### Как определяется уровень

1. **Начальный выбор** — при онбординге: начинающий (1-3), средний (4-6), продвинутый (7-10)
2. **Влияние на промпты** — `getDifficultyParams()` масштабирует параметры генерации: диапазон метров, количество строк, уровень подсказок
3. **Адаптация** (план): LLM возвращает `difficultyAdjustment: 'up'|'down'|'same'`, но пока не применяется автоматически к профилю

## Промпт-архитектура

### Генерация упражнения

Каждый тип имеет специализированный промпт генерации (8 творческих + 4 дрили = 12 шаблонов). Все вызываются через общий `getGeneratePrompt(type, topicName, difficulty)`.

### Проверка ответа

- **6 специализированных промптов** — rhythm, rhyme, metaphor (имеют уникальные критерии)
- **Остальные** — используют `genericReviewPrompt` с общими критериями
- **REVIEW_JSON_FORMAT** — appended к каждому промпту проверки с явными примерами формата (предотвращает 3 известных варианта нарушения формата от LLM)

### Обработка ответов LLM

Движок содержит многоуровневую защиту:
1. Авто-распаковка вложенного JSON (7 возможных ключей-обёрток)
2. Конвертация snake_case → camelCase
3. Нормализация flat-полей (3 варианта: `scores.rhythm`, `rhythmScore`, `rhythm`)
4. Фоллбэк на встроенный отзыв при ошибках парсинга

## Встроенные упражнения

Всего **8 встроенных упражнений** (5 дрелей + 3 творческих). Работают без API-ключа.

Все встроенные упражнения сохраняются в IndexedDB для корректного подсчёта в статистике темы.

| Тип | Кол-во | Сложности |
|-----|--------|-----------|
| rhythm | 2 | 2, 3 |
| rhyme | 1 | 2 |
| syllable_count | 2 | 1, 2 |
| stress_pattern | 1 | 1 |
| rhyme_match | 1 | 1 |
| line_builder | 1 | 2 |
| metaphor, constraint, deconstruction, phonetics, prose_to_poetry, anti_cliche | 0 | — |

## Анализ текста в реальном времени

Встроен в ExerciseScreen для творческих упражнений. Работает на клиенте, без LLM.

| Функция | Описание |
|---------|----------|
| Разделение на слоги | Каждая гласная = новый слог, сонорные и й разделяют гласные |
| Цветовая подсветка | Ударные слоги — ember, безударные — dusk |
| Подсчёт слогов | По строкам и общий |
| Визуализация ударений | Паттерн ТА/та |
| Детекция размера | 9 метров: хорей, ямб, дактиль, амфибрахий, анапест, пеон I–IV |
| Построчный анализ | Размер и уровень доверия для каждой строки |
| Общий размер | Определяется по большинству строк |
| Копирование анализа | Кнопка «Копировать анализ» |
