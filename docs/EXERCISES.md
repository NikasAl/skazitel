# Система упражнений Сказителя

## Обзор

Упражнения — основа Сказителя. Каждое упражнение тренирует конкретный навык стихосложения. Упражнения генерируются через LLM на основе темы пользователя, либо берутся из встроенного набора (офлайн-режим).

### Цикл взаимодействия

```
1. Пользователь выбирает/вводит тему
   ↓
2. LLM анализирует тему, определяет доступные образы
   ↓
3. Система предлагает упражнение подходящего типа и сложности
   ↓
4. Пользователь выполняет задание (вводит текст)
   ↓
5. LLM проверяет ответ по критериям упражнения
   ↓
6. Фидбек: что хорошо, что можно улучшить, конкретные советы
   ↓
7. Начисление XP, обновление стрика, проверка условия уровня
   ↓
8. Каждое 5-е упражнение — «Творческая сессия» (без оценки)
```

## Типы упражнений

### 1. Ритмический тренажёр (`rhythm`)

**Навык:** метрическое чутьё, чувство размера, подсчёт слогов.

**Механика:** Пользователь получает прозаический текст на свою тему и должен переложить его в заданный стихотворный размер с ограничениями по количеству строк и слогов.

**Пример с темой «осенний дождь»:**
```
Задание: Перепиши этот текст ямбом:
«Дождь барабанит по крыше, ветер качает деревья,
 листья падают в лужи»

Твой ответ должен содержать 4 строки по 8 слогов.
```

**Промпт-шаблон для генерации (system + user):**
```markdown
## Системный промпт

Ты — наставник по стихосложению. Создай упражнение типа «Ритмический тренажёр».

## Контекст
- Тема пользователя: {{topic}}
- Уровень сложности: {{difficulty}} (1-10)
- Предыдущие успехи: {{successRate}}%

## Задача
1. Напиши 2-3 прозаических предложения на тему {{topic}}.
   Текст должен быть связным, живым, содержать конкретные образы.
2. Выбери стихотворный размер, подходящий для уровня:
   - Уровни 1-3: ямб или хорей, 4 стопы
   - Уровни 4-6: дактиль, амфибрахий, 5 стоп
   - Уровни 7-10: анапест, смешанные размеры, вольный стих
3. Укажи количество строк (4–8) и слогов в строке.

## Формат ответа (JSON)
{
  "instruction": "Перепиши этот текст {{meter}}:\n«{{prose}}»\n\nТвой ответ должен содержать {{lines}} строк по {{syllables}} слогов.",
  "meter": "ямб",
  "syllableCount": 8,
  "expectedLines": 4,
  "proseText": "...",
  "examples": ["Пример первой строки с правильным ритмом (не полный ответ)"],
  "successCriteria": [
    "Строки написаны в указанном размере",
    "Каждая строка содержит указанное количество слогов",
    "Смысл исходного текста сохранён"
  ],
  "hints": [
    "Ямб — ударение на чётных слогах: та-ТА-та-ТА-та-ТА-та-ТА",
    "Считай слоги на пальцах, отбивай ритм рукой"
  ]
}
```

**Промпт-шаблон для проверки:**
```markdown

Ты — наставник по стихосложению. Проверь упражнение «Ритмический тренажёр».

## Формат ответа (JSON)
{
  "strengths": ["Что получилось хорошо (2-3 пункта)"],
  "weaknesses": ["Конкретные проблемы (2-3 пункта)"],
```

### 2. Рифмический конструктор (`rhyme`)

**Навык:** поиск рифм, построение схем рифмовки, точные и неточные рифмы.

**Механика:** Пользователю даются ключевые слова из его темы. Нужно составить четверостишие, используя все ключевые слова в позициях рифмующихся окончаний по заданной схеме.

**Пример:**
```
Ключевые слова: окно, стекло, туман, обман
Схема рифмовки: abab

Составь четверостишие используя ВСЕ эти слова
в позиции рифмующихся окончаний строк.
```

**Промпт-шаблон для генерации:**
```markdown
## Задача
Для темы {{topic}} подбери 4-6 слов которые:
- Семантически связаны с темой
- Образуют 2-3 рифмующиеся пары (точные или неточные)
- Разной сложности (от банальных до неочевидных)
- Учитывают уровень {{difficulty}}

Выбери схему рифмовки:
- Уровни 1-3: aabb (паровая)
- Уровни 4-6: abab (перекрёстная)
- Уровни 7-10: abba (кольцевая) или свободная

## Формат ответа (JSON)
{
  "instruction": "Ключевые слова: {{keywords}}\nСхема рифмовки: {{scheme}}\n\nСоставь четверостишие используя ВСЕ эти слова в позиции рифмующихся окончаний строк.",
  "keywords": ["окно", "стекло", "туман", "обман"],
  "rhymeScheme": "abab",
  "examples": ["Первая строка должна заканчиваться словом «окно»..."],
  "successCriteria": [
    "Все ключевые слова использованы",
    "Слова стоят в правильных позициях по схеме",
    "Строки образуют связный текст"
  ],
  "hints": ["abab: 1-я и 3-я строки рифмуются, 2-я и 4-я — между собой"]
}
```

### 3. Метафорический мост (`metaphor`)

**Навык:** построение неочевидных метафор, связь далёких понятий через общий признак.

**Механика:** Пользователь получает набор визуальных образов, связанных с темой, и должен написать двустишие на каждый образ, создавая метафорическую связь.

**Пример:**
```
Тема: «одиночество в большом городе»

Задание: Найди метафоры через образы:
- заброшенная станция метро
- сломанный светофор
- пустая витрина
- дождевая капля на стекле
- одинокий фонарь

Напиши по двустишию на каждую метафору.
```

**Промпт-шаблон для генерации:**
```markdown
## Задача
Для темы {{topic}} подбери 5 образов которые:
- Визуально яркие и конкретные
- Имеют параллели с темой (но неочевидные)
- Не являются клише
- Подходят для уровня {{difficulty}}

Для каждого образа напиши короткий пример двустишия (не полный ответ, а направление мысли).

## Формат ответа (JSON)
{
  "instruction": "Тема: «{{topic}}»\n\nНайди метафоры через образы:\n{{imageList}}\n\nНапиши по двустишию на каждую метафору.",
  "images": ["заброшенная станция метро", "сломанный светофор", ...],
  "targetEmotion": "одиночество",
  "examples": ["Станция — как пустое сердце города, поезда ушли..."],
  "successCriteria": [
    "Каждый образ раскрыт через метафору",
    "Метафоры неочевидные (не клише)",
    "Образы связаны общей темой"
  ]
}
```

### 4. Ограничение как искусство (`constraint`)

**Навык:** работа с формальными ограничениями, плотность текста, точность формулировок.

**Механика:** Жёсткие формальные ограничения (количество слов, части речи, времена глаголов и т.д.). Пользователь пишет стих, соблюдая все ограничения.

**Пример:**
```
Тема: «первая любовь»
Ограничения:
- Ровно 40 слов в стихотворении
- Ни одного прилагательного
- Только глаголы прошедшего времени
- Рифмовка: aabb
```

**Промпт-шаблон для генерации:**
```markdown
## Задача
Придумай для темы {{topic}} набор из 3-4 ограничений:
- Которые делают задачу сложной, но выполнимой
- Которые тренируют конкретный навык (плотность, точность, лаконичность)
- Адекватны уровню {{difficulty}}

Ограничения могут касаться:
- Количество слов / строк
- Запрещённые / обязательные части речи
- Время глаголов
- Обязательные звуки
- Схема рифмовки
- Другие формальные параметры

Объясни почему эти ограничения полезны.
```

### 5. Деконструкция мастера (`deconstruction`)

**Навык:** анализ поэтических приёмов, архитектура стихотворения, управление ожиданиями.

**Механика:** Пользователь получает отрывок известного стихотворения (близкого к теме) и должен написать свой вариант, используя те же приёмы.

**Пример:**
```
Прочитай как Бродский пишет о «расставании»:
[отрывок]

Заметь приёмы:
- Анжамбеман (перенос)
- Смешение высокого и низкого стиля
- Повторы

Напиши своё четверостишие на тему «{{topic}}»
используя минимум 2 из этих приёмов.
```

**Промпт-шаблон для генерации:**
```markdown
## Задача
Для темы {{topic}} найди отрывок из известного стихотворения
(русской или зарубежной поэзии), который тематически близок.

Выдели 3-4 конкретных поэтических приёма в этом отрывке:
- Анжамбеман, перенос смысла
- Контраст высокого и низкого стиля
- Повторы (анафора, эпифора)
- Аллитерация
- Оксюморон
- И т.д.

Объясни кратко как работает каждый приём.

## Формат ответа (JSON)
{
  "instruction": "Прочитай отрывок:\n«{{poemExcerpt}»\n\nПриёмы: {{techniques}}\n\nНапиши своё четверостишие на тему «{{topic}}» используя минимум 2 из этих приёмов.",
  "masterPoem": "...",
  "masterAuthor": "...",
  "techniques": ["анжамбеман", "смешение стилей"],
  "examples": []
}
```

### 6. Фонетический этюд (`phonetics`)

**Навык:** фонетическая музыкальность, звукопись, аллитерация, ассонанс.

**Механика:** Написать четверостишие, где преобладают определённые звуки для создания конкретного настроения.

**Пример:**
```
Тема: «зимний лес»
Задание: Напиши четверостишие где преобладают звуки [с], [ш], [щ]
для создания ощущения шелеста снега.
Минимум 8 слов с этими звуками.
```

---

### 7. Перевод прозы в поэзию (`prose_to_poetry`)

**Навык:** упаковка смысла в форму, сокращение без потери, ритмическая структура.

**Механика:** Низкий порог входа. Пользователь пишет прозаический текст, затем LLM помогает найти в нём поэтические элементы и предлагает превратить в стихи. Подходит для начинающих.

**Пример:**
```
Напиши честный текст о теме «{{topic}}» (200-300 слов).
Не думай о стихах, просто пиши как чувствуешь.

Затем мы вместе найдём в нём:
- Ритмические паттерны
- Образы которые можно усилить
- Фразы которые можно сократить
```

---

### 8. Анти-клише (`anti_cliche`)

**Навык:** оригинальность мышления, поиск неочевидных образов, избегание шаблонов.

**Механика:** Написать стих на тему, избегая очевидных образов и слов. LLM даёт список запрещённых клише.

**Пример:**
```
Тема: «закат»
Запрещённые слова: солнце, небо, вечер, красный, оранжевый,
                    уходить, спать, горизонт, догорать

Найди 5 неожиданных образов для заката и напиши стих.
```

**Промпт-шаблон для генерации:**
```markdown
## Задача
Для темы {{topic}} составь список из 10-15 клише:
- Слова-штампы (очевидные ассоциации)
- Банальные образы
- Затёртые рифмы
- Шаблонные метафоры

Это тренирует оригинальность мышления.
Список должен быть достаточно большим, чтобы создать настоящий вызов,
но не настолько, чтобы сделать задачу невыполнимой.
```

## Система сложности (1-10)

### Группы уровней

| Уровни | Название | Характеристика |
|--------|----------|---------------|
| **1-2** | Освоение формы | Простые рифмы, базовые размеры (4-стопный ямб), темы с очевидными образами, много подсказок и примеров |
| **3-4** | Развитие техники | Составные рифмы, чередование размеров, более абстрактные темы, меньше подсказок |
| **5-6** | Мастерство формы | Слантовые рифмы, сложные размеры (амфибрахий, анапест), метафорические темы, жёсткие ограничения |
| **7-8** | Свобода в рамках | Внутренние рифмы, полиритмия, концептуальные темы, минимум указаний |
| **9-10** | Создание нового | Эксперименты с формой, слом ожиданий, философские темы, полная свобода с самоконтролем |

### Как определяется уровень

1. **Начальный выбор** — при онбординге пользователь выбирает: начинающий (уровни 1-3), средний (4-6), продвинутый (7-10)
2. **Адаптация** — после 3 успешных упражнений подряд (overall score >= 80) сложность повышается на 1. После 3 неудачных (score < 40) — понижается на 1
3. **Влияние на промпты** — уровень передаётся в LLM-промпт, и провайдер адаптирует формулировку, количество подсказок и строгость ограничений

### Ограничения по сложности для типа упражнения

| Тип упражнения | Ур. 1-2 | Ур. 3-4 | Ур. 5-6 | Ур. 7-8 | Ур. 9-10 |
|----------------|---------|---------|---------|---------|---------|
| rhythm | 4-стопный ямб, 4 строки | 5 стоп, разные размеры | Смешанные размеры | Полиритмия | Вольный стих с ограничениями |
| rhyme | aabb, точные рифмы | abab, неточные рифмы | abba, составные | Внутренние рифмы | Свободная схема |
| metaphor | 3 очевидных образа | 5 образов, неочевидные | 5 образов + связь | 7 образов + концепция | Открытый поиск |
| constraint | 1-2 ограничения | 2-3 ограничения | 3-4, жёсткие | 4-5, комбинированные | Самоограничение |
| deconstruction | 1 приём | 2 приёма | 3 приёма | Комбинация приёмов | Свой стиль |
| phonetics | 1 звук, 5 слов | 2 звука, 8 слов | 3 звука, 10 слов | Звук + настроение | Полная звукопись |
| prose_to_poetry | 200 слов, много помощи | 200 слов, меньше помощи | 150 слов, мин. помощь | 100 слов, нет помощи | Свободная форма |
| anti_cliche | 5 запрещённых слов | 10 запрещённых | 15 + запрещённые образы | 20 + запрещённые темы | Полная оригинальность |

## Творческие сессии

Каждое 5-е упражнение — творческая сессия без оценки, XP и ограничений. Это защищает пользователя от превращения в «оптимизатора под алгоритм».

```
Тема: {{topic}} (та же, что тренировали)
Задание: Просто напиши что чувствуешь.
Нет ограничений, нет оценки, нет XP.
Если хочешь — попроси ИИ разобрать текст.
```

**Правила творческой сессии:**
- Нет оценки и баллов
- XP начисляется фиксированно (15) за сам факт участия
- Пользователь может попросить ИИ-разбор текста (опционально)
- Текст сохраняется в разделе «Мои стихи»
- Счётчик упражнений до следующей творческой сессии сбрасывается

## Промпт-архитектура

### Главный системный промпт (для всех запросов)

```markdown
Ты — наставник по стихосложению. Твоя задача — НЕ писать стихи за
пользователя, а помогать ему развивать навыки.

## Правила
1. Объясняй какую технику мы тренируем
2. Давай чёткое задание с ограничениями
3. Предоставляй 1-2 примера (но не полный ответ)
4. Указывай критерии успеха
5. При проверке: сначала strengths, затем weaknesses, потом suggestions
6. НИКОГДА не переписывай текст за пользователя

## Формат
Возвращай ответы в формате JSON.
```

### Промпт для генерации упражнения

Структура:
1. Системный промпт (наставник)
2. Контекст: тема, уровень, история успехов
3. Инструкция: тип упражнения и параметры генерации
4. Формат ответа: JSON-схема конкретного типа

### Промпт для проверки ответа

Структура:
1. Системный промпт (наставник)
2. Исходное задание (exercise)
3. Ответ пользователя
4. Критерии проверки (специфичные для типа)
5. Формат ответа: JSON с scores, strengths, weaknesses, suggestions

### Промпт для творческой сессии

Структура:
1. Системный промпт (более мягкий, без оценок)
2. Контекст: тема, прогресс пользователя
3. Инструкция: предложить свободное письмо
4. Формат: свободный текст (не JSON)

---

## Механизм «тема слишком сложная»

Если тема пользователя слишком абстрактная или специфичная для генерации качественных упражнений, LLM может вернуть флаг:

```json
{
  "error": "topic_too_complex",
  "message": "Тема слишком абстрактна для ритмических упражнений. Попробуй сформулировать её более конкретно.",
  "suggestedAlternatives": [
    "конкретная ситуация из темы",
    "ассоциация с темой"
  ]
}
```

Пользователь видит сообщение с предложением уточнить тему или выбрать из предложенных альтернатив.
